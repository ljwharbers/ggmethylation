# Plan A1: Strand Coloring

**Goal:** Color read bars (`geom_segment`) by strand (`+`/`-`) as an opt-in parameter, with defined behaviour when group coloring is already active.

**Affected files:**
- `R/plot_methylation.R` — all changes live here
- `man/plot_methylation.Rd` — regenerated by `devtools::document()`
- `R/read_methylation.R` — no changes needed; `strand` is already in `$reads`

---

## New parameters

Add to the `plot_methylation()` signature after `dot_size`:

```r
colour_strand = FALSE,
strand_colours = c("+" = "#4393C3", "-" = "#D6604D"),
```

| Parameter | Type | Default | Description |
|---|---|---|---|
| `colour_strand` | logical | `FALSE` | When `TRUE`, read bars are coloured by strand. Ignored when data is grouped (group colour takes precedence). |
| `strand_colours` | named character | blue/red | Named vector with `"+"` and `"-"` entries. Only used when `colour_strand = TRUE` and ungrouped. |

Default `FALSE` preserves current behaviour. Blue (`#4393C3`) for forward and red-orange (`#D6604D`) for reverse is a widely-used convention (ColorBrewer RdBu).

### Step 2 — Add input validation

After the existing `inherits` check:

```r
if (!is.logical(colour_strand) || length(colour_strand) != 1L) {
  stop("'colour_strand' must be a single logical value.", call. = FALSE)
}
if (isTRUE(colour_strand) && !all(c("+", "-") %in% names(strand_colours))) {
  stop("'strand_colours' must be a named vector with '+' and '-' entries.", call. = FALSE)
}
if (isTRUE(colour_strand) && !is.null(data$group_tag)) {
  message("'colour_strand = TRUE' is ignored when reads are grouped; group colour takes precedence.")
}
```

### Step 3 — Replace fixed-colour `geom_segment` in the ungrouped path

Replace the current `geom_segment` with a conditional block:

```r
if (isTRUE(colour_strand)) {
  p_top <- ggplot2::ggplot() +

      data = data$reads,

      data = sites_plot,

    ggplot2::scale_colour_gradient(
      low = colour_low, high = colour_high,
      limits = c(0, 1), name = "Modification\nprobability"
    )
} else {
  # current code unchanged
  p_top <- ggplot2::ggplot() +
    ggplot2::geom_segment(
      data = data$reads,
      ggplot2::aes(x = .data$start, xend = .data$end,
                   y = .data$lane, yend = .data$lane),
      linewidth = 2, colour = "#B0BEC5", lineend = "round"
```

## Interaction table

| Scenario | Bar colour | Colour scales in `p_top` |
|---|---|---|
| Ungrouped, `colour_strand = FALSE` | Fixed `"#B0BEC5"` (unchanged) | 1: mod prob gradient |
| Ungrouped, `colour_strand = TRUE` | Mapped to `strand` | 2: strand manual + mod prob gradient (via `new_scale_colour`) |
| Grouped, `colour_strand = FALSE` | Mapped to `group` | 2: group discrete + mod prob gradient (current) |
| Grouped, `colour_strand = TRUE` | Mapped to `group` (message fired) | 2: group discrete + mod prob gradient (unchanged) |

---

## Edge cases

1. **Reads with `"*"` strand** — Rsamtools returns `"*"` for unstranded reads. `strand_colours` does not include `"*"` by default, so bars will be invisible (`NA` colour). Document this; users can add `"*" = "grey70"` if needed.
2. **All reads same strand** — works; legend shows one entry.
3. **`strand_colours` missing a key** — caught by Step 2 validation before any plotting.
4. **Empty `data$reads`** — existing early-return fires before strand logic; no change needed.
5. **SNV dashed vline** — appended to `p_top` after the conditional block; works identically in both branches.

## Testing approach

1. **Default (unchanged)** — `plot_methylation(md)`: confirm `geom_segment` uses static `colour = "#B0BEC5"`, no strand scale present.
2. **`colour_strand = TRUE`** — confirm `geom_segment` maps `colour = ~strand`; confirm `p_top$scales` contains `ScaleManual` (strand) and `ScaleContinuous` (mod prob).
3. **Custom `strand_colours`** — pass `strand_colours = c("+" = "green", "-" = "purple")`; confirm the `ScaleManual$palette` maps to those values.
4. **Grouped + `colour_strand = TRUE`** — `expect_message(plot_methylation(md_grouped, colour_strand = TRUE))`, and confirm bar mapping is still `group`.
5. **Invalid `strand_colours`** — `expect_error(plot_methylation(md, colour_strand = TRUE, strand_colours = c("+" = "blue")))`.
