## BAM Dependency Strategy

### Chosen approach: pre-computed RDS cache + `eval=FALSE` for data-loading chunks

**Rationale:**

The PacBio BAM lives at an absolute HPC path that is not portable, cannot be bundled in the package (gigabytes), and is not suitable for CRAN/Bioconductor build farms. Three options were considered:

1. **Reference the HPC BAM path directly with `eval=FALSE` on all chunks** — vignette renders but produces no inline output. Poor user experience; output figures are absent from the rendered HTML.
2. **Bundled tiny example BAM** — ideal for portability, but requires feature C11 (example data) to be implemented first and involves BAM + BAI files that add non-trivial package size. Deferred to a later milestone.
3. **Pre-computed RDS + pre-rendered plots (chosen)** — the author runs `read_methylation()` on the real BAM once, saves `methylation_data` objects to `inst/extdata/vignette_cache.rds`, and renders plots to `inst/extdata/vignette_plots/*.png`. Vignette code chunks that call `read_methylation()` are marked `eval=FALSE` and shown for illustration. Chunks that call `plot_methylation()` load the cached objects from `system.file()` and are `eval=TRUE`, producing real inline output without the BAM.

This strategy:
- Produces a fully rendered vignette with real plots on any machine that installs the package.
- Keeps the data-loading call visible and educational (shown but not executed).
- Avoids bundling large binary data.
- Is compatible with Bioconductor build farms.

**Limitation:** If the package API changes, the cache must be manually regenerated by running `data-raw/build_vignette_cache.R` (see step 8 below).

## Example Region

The BAM covers chr21 and chr22 from PTCL8 tumor PacBio HiFi data. Suggested region for the vignette:

```
chr21:34,500,000-34,510,000
```

This is a 10 kb window on chr21 in a region likely to have reasonable read depth from a tumor sample and to overlap known CpG-dense loci. When generating the cache (step 8), run `read_methylation()` over this region and inspect the output. If read count is low (<20) or methylation sites are sparse, shift the window or widen to 20 kb (e.g., `chr21:34,500,000-34,520,000`). Prefer a window with:

- At least 30–50 reads overlapping.
- A mix of methylated and unmethylated sites (mod_prob values spread across 0–1).
- HP tag present in reads (to demonstrate haplotype grouping).

For the SNV-based grouping example, identify an SNV position within the chosen window by inspecting `$reads$start`/`$reads$end` and cross-referencing with known variant calls, or simply choose a position that falls in reads carrying both a common reference and alternate base. If no clean SNV is available in the window, use a second nearby window for that section only.

## Implementation Steps

1. **Create the `vignettes/` directory** (if absent) and add `vignettes/ggmethylation.Rmd` with the YAML front matter:

   ```yaml
   ---
   title: "Visualising read-level methylation with ggmethylation"
   author: "Luuk Harbers"
   date: "`r Sys.Date()`"
   output: rmarkdown::html_vignette
   vignette: >
     %\VignetteIndexEntry{Visualising read-level methylation with ggmethylation}
     %\VignetteEngine{knitr::rmarkdown}
     %\VignetteEncoding{UTF-8}
   ---
   ```

   Do **not** add `BiocStyle` as a dependency at this stage. Plain `rmarkdown::html_vignette` is sufficient for an early-stage package and avoids an extra `Suggests` entry. Revisit if submitting to Bioconductor proper.

2. **Update `DESCRIPTION`** with the following additions:

   ```
   Suggests:
       testthat (>= 3.0.0),
       knitr,
       rmarkdown
   VignetteBuilder: knitr
   ```

   `knitr` and `rmarkdown` go in `Suggests` (not `Imports`) because they are only needed to build the vignette, not to use the package. Do not add `BiocStyle` unless opting in to Bioconductor submission (add that note as a TODO comment).

3. **Create `data-raw/build_vignette_cache.R`** — a standalone R script (not part of the installed package) that:
   - Sources the package from the local clone (`devtools::load_all()`).
   - Calls `read_methylation()` for the chosen region with three configurations:
     - `meth_basic`: ungrouped
     - `meth_hp`: `group_tag = "HP"`
     - `meth_snv`: `snv_position = <pos>, ref_base = <ref>, alt_base = <alt>`
   - Saves all three objects to a named list in `inst/extdata/vignette_cache.rds` via `saveRDS()`.
   - Optionally renders and saves the four key plots as PNG files to `inst/extdata/vignette_plots/` using `ggplot2::ggsave()` (not strictly needed if plots are rendered at build time from cached objects, but useful as a visual record).
   - This script must be run manually by the maintainer whenever the API or BAM data changes. It is never called during `R CMD check`.

4. **Create `inst/extdata/` directory** (if absent) and run `data-raw/build_vignette_cache.R` once on the HPC against the real BAM to produce `vignette_cache.rds`. Confirm the file is small enough (< 1 MB typically for a few hundred reads).

5. **Add `data-raw/` to `.Rbuildignore`** so the cache-building script is not included in the installed package:

   ```
   ^data-raw$
   ```

6. **Write the vignette body** following the section outline above. Key chunk options to use:
   - Data-loading chunks (those calling `read_methylation()`): `eval=FALSE, echo=TRUE` — code is shown but not executed.
   - Cache-loading chunks (those calling `readRDS(system.file(...))`): `eval=TRUE, echo=FALSE` — silently loads objects; user does not need to see this plumbing.
   - All `plot_methylation()` and data-inspection chunks: `eval=TRUE, echo=TRUE` — shown and executed.
   - Use `fig.width=8, fig.height=5` (or appropriate) for plot chunks; adjust per section.
   - Set global chunk options in the setup chunk:
     ```r
     knitr::opts_chunk$set(
       collapse = TRUE,
       comment = "#>",
       fig.width = 8,
       fig.height = 5
     )
     ```

7. **Add a `.Rbuildignore` entry** (if not already present) to exclude any accidental references to the absolute BAM path from build tooling. This is a precaution — if the BAM path appears only inside `eval=FALSE` chunks, it does not affect the build.

8. **Test the vignette locally** by running:
   ```r
   devtools::build_vignettes()
   ```
   Confirm the HTML renders without errors and all inline plots are populated from the cache.

9. **Run `R CMD check` with vignettes** to confirm the vignette passes:
   ```bash
   R CMD build .
   R CMD check --as-cran ggmethylation_*.tar.gz
   ```
   The check should complete in under 30 seconds for vignette evaluation (no BAM I/O, only RDS load + ggplot2 rendering).

## Edge Cases and Build Considerations

### Long-running chunks
All `read_methylation()` calls are `eval=FALSE`. The only computation at build time is loading an RDS (< 1 s) and rendering ggplot2 plots (a few seconds each). Total vignette build time should be well under 60 seconds — comfortably within Bioconductor build farm limits.

### Cache staleness
If `read_methylation()` output format changes (e.g., new columns added to `$reads` or `$sites`), the cached RDS must be regenerated. Add a comment at the top of `data-raw/build_vignette_cache.R` documenting the package version and BAM path used, so the regeneration context is always clear.

### `inst/extdata/vignette_cache.rds` size
A `methylation_data` object for 3 x ~100-read regions is typically 100–500 KB. If it exceeds 1 MB, consider downsampling `max_reads` further (e.g. to 50) for the cached objects, or serializing with `compress = "xz"` in `saveRDS()`.

### Bioconductor build farm portability
`system.file("extdata", "vignette_cache.rds", package = "ggmethylation")` will resolve correctly on any machine where the package is installed, because `inst/extdata/` is copied into the installed package tree. This is the standard pattern for vignette data in Bioconductor packages.

### `html_vignette` vs `html_document`
`rmarkdown::html_vignette` produces a compact HTML without external CSS/JS dependencies, which is required for CRAN compatibility. Do not switch to `html_document` unless opting out of CRAN submission.

### Bioconductor submission (future)
If the package is submitted to Bioconductor, add `BiocStyle` to `Suggests` and update the YAML output field to `BiocStyle::html_document`. BiocStyle adds Bioconductor-standard formatting and a proper package version header. This is a one-line YAML change and one `Suggests` addition — defer until submission is planned.

### Figure file sizes in the vignette HTML
With `eval=TRUE` plot chunks, knitr embeds plots as base64 PNG in the HTML by default with `html_vignette`. For four plots at 8x5 inches, the resulting HTML should be 1–3 MB — acceptable. If size is a concern, reduce `dpi` in chunk options (e.g., `dpi=96`).

### SNV example fallback
If no clean SNV is identifiable in the primary window `chr21:34,500,000-34,510,000`, the SNV section can use a second window on chr22. Document the region used in `data-raw/build_vignette_cache.R` with a comment.

### `.Rbuildignore` for `data-raw/`
The line `^data-raw$` must already be added to `.Rbuildignore` if following the standard `usethis` workflow. Verify this is present before submitting to CRAN/Bioconductor.
